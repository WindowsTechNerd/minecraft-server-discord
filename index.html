About this EFI

The goal of this guide is not just to give you a pre‑built EFI, but to show you how to assemble one yourself. That way, when macOS, OpenCore, or your kexts update, you understand what to change and why.

Step‑by‑step EFI build guide

Follow these steps in order. Don’t copy random EFIs from the internet—build yours intentionally so you can debug it when something breaks.

  1. 1. Prepare BIOS settings
    Enter BIOS and configure:
    • Disable Secure Boot
    • Set SATA mode to AHCI
    • Enable UEFI boot (no Legacy)
    • Disable Intel RST / RAID if present
    Tip: Take photos of your original BIOS settings so you can revert if needed.
  2. 2. Create the macOS installer
    On a real Mac or working Hackintosh: 
    # Example (Big Sur)
sudo /Applications/Install\ macOS\ Big\ Sur.app/Contents/Resources/createinstallmedia \
  --volume /Volumes/MyUSB
    Then mount the USB’s EFI partition and create an EFI folder on it.
  3. 3. Add OpenCore base files
    Download the latest OpenCore release matching your CPU generation, then copy: 
    EFI/
  BOOT/
    BOOTx64.efi
  OC/
    OpenCore.efi
    Drivers/
    Kexts/
    ACPI/
    config.plist
    Start from the sample config for your platform (Skylake) from the official OpenCore guide.
  4. 4. Add required kexts
    At minimum, you’ll want:
    • Lilu.kext
    • VirtualSMC.kext (+ optional sensors plugins)
    • WhateverGreen.kext
    • AppleALC.kext (for Conexant audio)
    • VoodooPS2Controller.kext (keyboard/trackpad)
    Place them in: 
    EFI/OC/Kexts/
  5. 5. Configure Wi‑Fi kexts (Intel)
    For the Intel 8260 card, you can choose between:
    • AirportItlwm.kext – integrates with native Wi‑Fi menu
    • itlwm.kext – often more stable, used with HeliPort app
    Download from the official OpenIntelWireless releases and place your chosen kext in: 
    EFI/OC/Kexts/
    Make sure you pick the correct build for your macOS version (e.g. Big Sur vs Ventura).
  6. 6. Configure Bluetooth kexts
    For Intel Bluetooth, you’ll typically use:
    • IntelBluetoothFirmware.kext
    • IntelBluetoothInjector.kext (for older macOS)
    Place them in EFI/OC/Kexts and enable them in config.plist under Kernel → Add.
  7. 7. Add ACPI patches
    Use SSDTs generated or recommended for:
    • EC (Embedded Controller)
    • USB mapping
    • Power management (if applicable)
    Place them in: 
    EFI/OC/ACPI/
    and reference them in config.plist → ACPI → Add.
  8. 8. Configure SMBIOS
    For this machine, a common choice is: 
    SystemProductName: MacBookPro13,1
    Use GenSMBIOS to generate:
    • Serial Number
    • Board Serial (MLB)
    • System UUID
    • ROM (usually your MAC address)
    Never reuse someone else’s SMBIOS. Always generate your own to avoid Apple ID / iCloud issues.
  9. 9. Install macOS
    Boot from the USB, select the OpenCore entry, and install macOS to your target drive. After installation, mount the internal disk’s EFI partition and copy your working EFI from the USB to the internal drive.
  10. 10. Refine and debug
    Once macOS boots:
    • Test Wi‑Fi, Bluetooth, audio, sleep, and battery reporting
    • Check logs if something fails (OpenCore debug builds can help)
    • Update kexts and OpenCore periodically following the same structure

Post‑install notes

After you have a stable system, keep a backup of your working EFI and avoid updating everything at once. Change one thing at a time so you always know what broke—or what fixed—your setup.

Streaming & DRM

Safari DRM is often broken on Hackintosh systems with Intel Wi‑Fi. If Apple TV+, Netflix, or Prime Video don’t work in Safari, use a Chromium‑based browser (Chrome, Edge, Brave, etc.).
